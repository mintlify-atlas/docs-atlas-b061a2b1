---
title: "mise-env-fnox"
description: "A mise env plugin that loads secrets from fnox into your development environment as environment variables."
---

mise-env-fnox bridges [mise](https://mise.jdx.dev) and [fnox](https://github.com/jdx/fnox), automatically injecting your fnox-managed secrets as environment variables whenever mise activates your project environment. No manual `export` commands, no plaintext secrets in `.env` files.

<Warning>
  The author of this plugin notes uncertainty about whether it's a good idea — fnox was built as a separate CLI for a reason. Consider whether this integration fits your workflow before adopting it.
</Warning>

## Quick setup

<Steps>
  <Step title="Add the plugin to mise.toml">
    Declare the plugin and activate it in your project's `mise.toml`:

    ```toml mise.toml
    [plugins]
    fnox-env = "https://github.com/jdx/mise-env-fnox"

    [env]
    _.fnox-env = { tools = true }
    ```
  </Step>
  <Step title="Ensure fnox is configured">
    Make sure a `fnox.toml` exists in your project directory (or a parent directory) with your secrets configured.
  </Step>
  <Step title="Activate your environment">
    Run any mise command — secrets are automatically exported as environment variables:

    ```bash
    mise env
    ```
  </Step>
</Steps>
